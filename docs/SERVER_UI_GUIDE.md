# 服务器管理 UI 使用指南

## 访问入口

打开 **设置** 应用 → 点击侧边栏的 **服务器管理** 标签

## 界面概览

### 主界面

服务器管理界面包含：
- **标题栏**：显示"服务器管理"标题和"添加服务器"按钮
- **服务器列表**：展示所有已配置的服务器卡片
- **空状态**：当没有服务器时显示引导信息

### 服务器卡片

每个服务器卡片显示：
- **服务器名称**：如"生产服务器"
- **标签**：如"生产"、"Web"、"前端"
- **地址和端口**：如 `192.168.1.100:22`
- **用户名**：如 `root`
- **认证方式**：密码或密钥
- **描述**：可选的服务器描述
- **添加时间**：服务器添加的时间
- **操作按钮**：
  - 🧪 测试连接
  - ✏️ 编辑
  - 🗑️ 删除

## 添加服务器

### 步骤

1. 点击右上角的 **添加服务器** 按钮
2. 填写服务器信息：

#### 必填字段

- **服务器名称**：给服务器起一个易于识别的名称
  - 示例：`生产服务器`、`开发环境`、`测试机`

- **主机地址**：服务器的 IP 地址或域名
  - 示例：`192.168.1.100`、`server.example.com`

- **端口**：SSH 端口（默认 22）
  - 示例：`22`、`2222`

- **用户名**：SSH 登录用户名
  - 示例：`root`、`ubuntu`、`admin`

- **认证方式**：选择密码认证或密钥认证
  - **密码认证**：简单但安全性较低
  - **密钥认证**：推荐使用，安全性高

#### 密码认证

选择"密码认证"后，需要填写：
- **密码**：SSH 登录密码

#### 密钥认证（推荐）

选择"密钥认证（推荐）"后，需要填写：
- **私钥**：SSH 私钥内容（PEM 格式）
  ```
  -----BEGIN RSA PRIVATE KEY-----
  MIIEpAIBAAKCAQEA...
  -----END RSA PRIVATE KEY-----
  ```
- **私钥密码**（可选）：如果私钥有密码保护

#### 可选字段

- **描述**：服务器的简短描述
  - 示例：`主要 Web 服务器`、`数据库服务器`

- **标签**：用逗号分隔的标签列表
  - 示例：`生产, Web, 前端`
  - 标签会显示在服务器卡片上

3. 点击 **添加** 按钮完成添加

### 示例：添加密码认证服务器

```
服务器名称：生产服务器
主机地址：192.168.1.100
端口：22
用户名：root
认证方式：密码认证
密码：your_password
描述：主要 Web 服务器
标签：生产, Web
```

### 示例：添加密钥认证服务器

```
服务器名称：开发服务器
主机地址：dev.example.com
端口：22
用户名：ubuntu
认证方式：密钥认证（推荐）
私钥：
-----BEGIN RSA PRIVATE KEY-----
MIIEpAIBAAKCAQEA...
-----END RSA PRIVATE KEY-----
私钥密码：（如果有）
描述：开发环境
标签：开发, 测试
```

## 测试连接

### 步骤

1. 在服务器卡片上找到 🧪 测试按钮
2. 点击测试按钮
3. 等待测试结果（按钮会显示加载动画）
4. 查看测试结果弹窗：
   - **成功**：显示"连接成功！耗时 XXXms"
   - **失败**：显示"连接失败：错误信息"

### 常见错误

- **连接超时**：检查服务器地址和端口是否正确
- **认证失败**：检查用户名和密码/私钥是否正确
- **网络不可达**：检查服务器是否在线，防火墙是否开放

## 编辑服务器

### 步骤

1. 在服务器卡片上找到 ✏️ 编辑按钮
2. 点击编辑按钮
3. 修改需要更改的字段：
   - 服务器名称
   - 主机地址
   - 端口
   - 用户名
   - 认证方式
   - 密码/私钥（可选）
   - 描述
   - 标签
4. 点击"保存"按钮
5. 服务器信息更新成功

### 密码/私钥更新

在编辑模式下：
- **密码/私钥字段显示"（留空则不修改）"**
- 如果不需要更改密码/私钥，留空即可
- 如果需要更改，填写新的密码/私钥
- 这样可以保护现有认证信息不被意外清空

### 示例：更新服务器描述

```
1. 点击编辑按钮
2. 修改描述字段：
   原描述：主要 Web 服务器
   新描述：主要 Web 服务器，运行 Nginx + Node.js
3. 密码字段留空（不修改）
4. 点击"保存"
```

### 示例：更换认证方式

```
1. 点击编辑按钮
2. 将认证方式从"密码认证"改为"密钥认证（推荐）"
3. 填写私钥内容
4. 点击"保存"
```

## 删除服务器

### 步骤

1. 在服务器卡片上找到 🗑️ 删除按钮
2. 点击删除按钮
3. 确认删除操作
4. 服务器将从列表中移除

**注意**：删除操作不可恢复，请谨慎操作。

## 使用场景

### 场景 1：管理多台服务器

添加多台服务器后，可以在列表中查看所有服务器的信息：
- 生产服务器（192.168.1.100）
- 开发服务器（dev.example.com）
- 测试服务器（test.example.com）

使用标签进行分类：
- 生产环境：`生产`、`Web`
- 开发环境：`开发`、`测试`
- 数据库服务器：`生产`、`数据库`

### 场景 2：快速测试连接

在部署前，先测试服务器连接：
1. 添加服务器
2. 点击测试连接
3. 确认连接成功后再进行部署

### 场景 3：安全管理

使用密钥认证替代密码认证：
1. 生成 SSH 密钥对
2. 将公钥添加到服务器
3. 在界面中使用私钥认证
4. 更安全，不需要记住密码

## 与 X 的配合

添加服务器后，X 可以使用这些服务器：

```
用户："帮我在生产服务器上部署应用"

X 会自动：
1. 从服务器列表中找到"生产服务器"
2. 使用 server.connect 连接
3. 执行部署命令
4. 上传文件
5. 启动应用
```

X 可以使用的服务器工具：
- `server.list` - 列出所有服务器
- `server.connect` - 连接到服务器
- `server.exec` - 执行命令
- `server.upload` - 上传文件
- `server.download` - 下载文件
- `server.disconnect` - 断开连接

定期测试服务器连接，确保配置正确：
- 添加服务器后立即测试

## 故障排除

### 问题 1：无法添加服务器

**现象**：点击"添加"按钮后提示错误

**解决**：
1. 检查必填字段是否都已填写
2. 检查端口是否为有效数字
3. 检查私钥格式是否正确（PEM 格式）

### 问题 2：测试连接失败

**现象**：点击测试按钮后提示"连接失败"

**解决**：
1. 检查服务器地址和端口是否正确
2. 检查用户名和密码/私钥是否正确
3. 检查服务器是否在线
4. 检查防火墙是否开放 SSH 端口
5. 检查私钥格式是否正确

### 问题 3：私钥格式错误

**现象**：添加服务器时提示"私钥格式错误"

**解决**：
确保私钥是 PEM 格式，应该类似：
```
-----BEGIN RSA PRIVATE KEY-----
MIIEpAIBAAKCAQEA...
...
-----END RSA PRIVATE KEY-----
```

如果是 OpenSSH 格式（`-----BEGIN OPENSSH PRIVATE KEY-----`），需要转换为 PEM 格式：
```bash
ssh-keygen -p -m PEM -f ~/.ssh/id_rsa
```

### 问题 4：列表不更新

**现象**：添加或删除服务器后列表没有更新

**解决**：
1. 刷新页面
2. 重新打开设置应用
3. 检查浏览器控制台是否有错误

## 安全建议

### 1. 不要在公共场所添加服务器

在公共场所（如咖啡厅、图书馆）使用时，避免添加服务器配置，因为可能被他人看到。

### 2. 定期更换密钥

定期更换 SSH 密钥，提高安全性：
1. 生成新的密钥对
2. 将新公钥添加到服务器
3. 更新界面中的私钥
4. 删除旧密钥

### 3. 使用强密码

如果使用密码认证，确保密码足够强：
- 至少 12 位
- 包含大小写字母、数字、特殊字符
- 不使用常见密码

### 4. 限制 SSH 访问

在服务器上配置防火墙，只允许特定 IP 访问 SSH 端口。

### 5. 使用非标准端口

将 SSH 端口从默认的 22 改为其他端口（如 2222），减少被扫描的风险。
