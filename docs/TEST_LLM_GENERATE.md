# llm.generate 真实化 — 可测试用例

用于验收「任务内 llm.generate 调用真实大模型」功能。

---

## 前置条件

1. **大模型已配置**  
   打开 **系统设置 → 大模型配置**：  
   - 至少添加一个提供商（如 OpenAI / Ollama / 自定义），并填写 Base URL；  
   - 若需 API Key，在对应提供商下填写；  
   - 在「聊天」模态下选择该提供商和模型。

2. **执行模式**  
   - 若选 **自动模式**：任务创建后会直接跑完所有步骤（含 llm.generate）。  
   - 若选 **审批模式**：前几步可能需在「任务时间线」里逐步批准，批准到「生成回复」后才会执行 llm.generate。

---

## 用例：Chat 域任务（含 llm.generate 步骤）

Chat 域任务模板包含三步：**分析用户意图**（llm.analyze）→ **检索相关资料**（search.web）→ **生成回复**（llm.generate）。  
下面用一条会命中 **chat** 域、且会创建任务的输入，专门验证第三步是否为真实 LLM 输出。

### 操作步骤

1. 打开 **AI 助手**（桌面或任务栏进入）。
2. 在输入框输入（任选其一即可）：
   - **推荐**：`分析一下什么是机器学习，用一两句话说明`
   - 或：`生成一段关于春天的简短描写，50字以内`
3. 发送。  
   - 系统会识别为「任务请求」（含关键词「分析」或「生成」），且因未命中办公/编程/智能体关键词，**域为 chat**。
4. 等待任务执行：
   - **自动模式**：等几秒直到任务完成。  
   - **审批模式**：到 **任务时间线** 中对该任务逐步「批准」，直到「生成回复」步骤执行完毕。
5. 在 **任务时间线** 中打开该任务，查看步骤详情。

### 预期结果

- 任务域为 **chat**，共 3 步：分析用户意图 → 检索相关资料 → 生成回复。
- 前两步为模拟输出（分析/检索的固定文案）。
- **第三步「生成回复」**：  
  - **若已配置 llmConfig（大模型）**：  
    - `output` 中的 `text` 应为**真实模型生成**的、与「机器学习」或「春天」相关的自然段，而不是占位文案 `[AI 生成内容] 基于: "..."`。  
  - **若未配置大模型**：  
    - `output.text` 仍为模拟文案：`[AI 生成内容] 基于: "分析一下什么是机器学习..."`（或你输入的前 50 字）。

### 如何快速区分「真实」与「模拟」

- **真实**：`text` 是一段完整、通顺的短句/段落，且和你的问题相关。  
- **模拟**：`text` 固定为 `[AI 生成内容] 基于: "你输入的前50个字"`。

## 可选：终端内创建任务

1. 打开 **终端**。
2. 输入：`ai 分析一下什么是机器学习，用一两句话说明`
3. 回车后任务会创建（同样会带上当前设置的 llmConfig）。
4. 到 **任务时间线** 查看该任务，验收「生成回复」步骤的 output 是否为真实 LLM 输出（同上）。

---

## 可选：Office 域（验证 file.write + 无 llm.generate）

Office 域没有 llm.generate，只有 file.write。若只想验证「任务能跑通 + 文件写入」，可用：

- 输入：`帮我生成本周工作周报`
- 预期：域为 **office**，最后一步 **file.write** 会在沙箱 `文档/` 下生成 `ai-output-*.txt`；可在 **文件管理器** 中打开该目录查看。

此用例不检验 llm.generate，仅作任务与 file 工具联调参考。

## 用例：生成并写入文件（Function Call）

当用户**明确要求把生成内容保存到文件**时，llm.generate 会通过 **function call** 调用 `file_write`，由模型决定是否写入；默认只返回文本，不写文件。

### 操作步骤

1. 打开 **AI 助手**，确保已配置大模型（支持 function call 的模型，如 gpt-4o、gpt-4、部分开源模型）。
2. 输入（任选其一）：
   - **推荐**：`分析一下什么是机器学习，用一两句话说明，并保存到 文档/机器学习简介.txt`
   - 或：`生成一段关于春天的简短描写并写入文件 文档/春天.txt`
3. 发送，等待任务执行完成（自动模式）或在任务时间线中批准到「生成回复」步骤。
4. 在 **任务时间线** 中查看该任务第三步「生成回复」的 output：应包含 `text` 和 `writtenFiles: ['文档/机器学习简介.txt']`（或对应路径）。
5. 打开 **文件管理器**，进入「文档」目录，确认对应文件存在且内容为模型生成的文本。

### 预期结果

- 模型返回了文本回复，且若用户要求「保存到文件」，则 **writtenFiles** 非空，沙箱内对应路径下出现新文件，内容与生成文本一致。
- 若用户**没有**要求保存到文件（例如只说「分析一下什么是机器学习」），则不应写入文件，output 中无 writtenFiles 或 writtenFiles 为空。
