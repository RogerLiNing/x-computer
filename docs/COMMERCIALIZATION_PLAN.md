# X-Computer 商业化上线计划

**创建日期**：2026-02-28  
**目标**：将 X-Computer 上线为收费 SaaS 服务  
**优先级**：P0（阻塞发布）

---

## 1. 需求概述

### 1.1 核心需求

1. **多语言支持**：界面和提示词支持中英文切换
2. **营销首页**：产品介绍、特性展示、定价说明
3. **付费体系**：试用 + 订阅付费，支持多种套餐

### 1.2 商业模式

- **免费试用**：7 天或限量使用（如 100 次 AI 调用）
- **付费订阅**：月付/年付，不同套餐提供不同配额
- **企业版**：私有部署或高级功能

## 2. 功能模块拆解

### 2.1 国际化（i18n）- R054

**目标**：界面和 AI 提示词支持中英文

#### 前端国际化

**技术方案**：
- 使用 `react-i18next` 或 `react-intl`
- 语言包结构：
  ```
  frontend/src/locales/
    en.json    # 英文
    zh-CN.json # 简体中文
  ```

**需要翻译的内容**：
- 应用名称、菜单、按钮
- 设置页面所有文本
- 错误提示、Toast 消息
- 应用描述（文件管理器、终端等）

**实现要点**：
- 在 `localStorage` 或用户配置中保存语言偏好
- 提供语言切换器（设置页 + 登录页）
- 默认根据浏览器语言自动选择

#### 后端/AI 提示词国际化

**需要支持的内容**：
- X 主脑系统提示词（`corePrompt.ts`）
- AI 助手提示词
- 工具描述（tool descriptions）
- 错误消息、通知内容

**实现方案**：
- 用户配置中增加 `preferredLanguage: 'en' | 'zh-CN'`
- 根据用户语言动态选择提示词模板
- 提示词文件结构：
  ```
  server/src/prompts/
    systemCore/
      corePrompt.en.ts
      corePrompt.zh-CN.ts
  ```

**优先级**：P0（必须）

### 2.2 营销首页 - R055

**目标**：专业的产品介绍页面，吸引用户注册

#### 页面结构

1. **Hero Section**（首屏）
   - 主标题："AI-Powered Autonomous Computer System"
   - 副标题：核心价值主张
   - CTA 按钮："Start Free Trial" / "Get Started"
   - 产品演示视频或动画

2. **Features Section**（特性展示）
   - AI 主脑自主运行
   - 办公应用套件
   - 小程序/小游戏创作
   - 多渠道通信（邮件、WhatsApp）
   - 远程服务器管理
   - 工作流自动化

3. **How It Works**（工作原理）
   - 3-4 步简单说明
   - 配图或动画演示

4. **Pricing**（定价）
   - 免费试用
   - 个人版
   - 专业版
   - 企业版

5. **Testimonials**（用户评价）
   - 可选，初期可省略

6. **Footer**（页脚）
   - 关于我们、联系方式
   - 隐私政策、服务条款
   - 社交媒体链接

#### 技术实现

**方案 A**：独立静态页面（推荐）
- 使用 Next.js 或纯 HTML/CSS/JS
- 部署到 CDN（Vercel、Cloudflare Pages）
- 优点：加载快、SEO 友好、独立部署

**方案 B**：集成到现有前端
- 在 React 应用中添加 `/landing` 路由
- 未登录时显示首页，已登录跳转桌面
- 优点：统一代码库、共享组件

**推荐**：方案 A（独立部署，更灵活）

#### 设计风格

- **现代简洁**：Mac/iOS 风格延续
- **深色主题**：与产品界面一致
- **响应式**：支持手机、平板、桌面
- **动画效果**：流畅的过渡和交互

**优先级**：P0（必须）

### 2.4 付费与订阅系统 - R057

**目标**：实现试用、订阅、配额管理

#### 订阅套餐设计

| 套餐 | 价格 | AI 调用次数 | 存储空间 | 并发任务 | 其他限制 |
|------|------|------------|---------|---------|---------|
| **免费试用** | $0 | 100 次 | 100MB | 1 | 7 天有效期 |
| **个人版** | $9.99/月 | 1000 次 | 1GB | 3 | - |
| **专业版** | $29.99/月 | 5000 次 | 10GB | 10 | 优先支持 |
| **企业版** | 定制 | 无限 | 无限 | 无限 | 私有部署 |

#### 技术实现

**支付集成**：
- **Stripe**（推荐）：国际支付、订阅管理
- **支付宝/微信支付**：中国市场
- **Paddle**：备选方案

**数据库设计**：

```sql
-- 订阅表
CREATE TABLE subscriptions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  plan_id TEXT NOT NULL,  -- 'trial' | 'personal' | 'pro' | 'enterprise'
  status TEXT NOT NULL,    -- 'active' | 'canceled' | 'expired' | 'past_due'
  current_period_start INTEGER NOT NULL,
  current_period_end INTEGER NOT NULL,
  stripe_subscription_id TEXT,
  stripe_customer_id TEXT,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 配额使用表
CREATE TABLE usage_records (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  resource_type TEXT NOT NULL,  -- 'ai_calls' | 'storage' | 'tasks'
  amount INTEGER NOT NULL,
  period_start INTEGER NOT NULL,
  period_end INTEGER NOT NULL,
  created_at INTEGER NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 套餐定义表
CREATE TABLE plans (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  display_name_en TEXT NOT NULL,
  display_name_zh TEXT NOT NULL,
  price_monthly INTEGER,  -- 美分
  price_yearly INTEGER,
  ai_calls_limit INTEGER,
  storage_limit INTEGER,  -- MB
  concurrent_tasks_limit INTEGER,
  features TEXT,  -- JSON array
  is_active INTEGER DEFAULT 1,
  created_at INTEGER NOT NULL
);
```

#### 配额管理

**限流策略**：
- AI 调用次数：每次 LLM 请求计数
- 存储空间：沙箱文件总大小
- 并发任务：同时运行的任务数
- 超限处理：友好提示升级套餐

**实现位置**：
- 中间件：`server/src/middleware/quotaCheck.ts`
- 在 API 路由中检查配额
- 超限时返回 402 Payment Required

**优先级**：P0（必须）

### 2.5 用户仪表板 - R058

**目标**：用户可查看订阅状态、使用情况、账单

#### 功能模块

1. **订阅管理**
   - 当前套餐显示
   - 升级/降级按钮
   - 取消订阅
   - 续费提醒

2. **使用统计**
   - AI 调用次数（本月/总计）
   - 存储空间使用
   - 任务执行次数
   - 图表可视化

3. **账单历史**
   - 历史订单列表
   - 发票下载
   - 支付方式管理

4. **账户设置**
   - 个人信息编辑
   - 密码修改
   - 邮箱验证状态
   - 危险操作（删除账户）

#### 实现方式

**前端**：
- 新增「账户」应用（`AccountApp`）
- 或在设置页面增加「订阅」标签

**后端 API**：
```
GET  /api/users/me/subscription      # 获取订阅信息
POST /api/users/me/subscription      # 创建/升级订阅
DELETE /api/users/me/subscription    # 取消订阅
GET  /api/users/me/usage             # 获取使用统计
GET  /api/users/me/invoices          # 获取账单列表
```

**优先级**：P1（高优先级）

#### 功能需求

1. **用户管理**
   - 用户列表、搜索、筛选
   - 查看用户详情
   - 手动调整配额
   - 封禁/解封用户

2. **订阅管理**
   - 订阅列表
   - 手动创建/修改订阅
   - 退款处理

3. **系统监控**
   - 总用户数、活跃用户
   - 收入统计
   - API 调用量
   - 错误日志

4. **内容管理**
   - 公告发布
   - 邮件模板编辑

#### 技术实现

**方案 A**：独立后台系统（推荐）
- 使用 React Admin 或 Ant Design Pro
- 独立部署，仅管理员可访问
- 调用主服务 API

**方案 B**：集成到主应用
- 在主应用中增加 `/admin` 路由
- 基于角色的访问控制（RBAC）

**推荐**：方案 A（安全性更好）

**优先级**：P2（中优先级）

### 5.1 开发成本

- 开发时间：6-7 周
- 如外包：$15,000 - $25,000

### 5.2 运营成本（月度）

| 项目 | 费用 | 说明 |
|------|------|------|
| 服务器 | $50-200 | 根据用户量 |

| SSL 证书 | 免费 | Let's Encrypt |
| 监控服务 | $0-50 | Sentry 免费额度 |

**预计月成本**：$100-400（初期）

### 5.3 收入预测

假设：
- 月活用户：100 人
- 付费转化率：10%
- 平均客单价：$19.99/月

**月收入**：100 × 10% × $19.99 = **$199.90**

**盈亏平衡点**：约 20 个付费用户

## 6. 风险与应对

### 6.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 支付集成复杂 | 延期 | 提前测试 Stripe API |
| 配额限制漏洞 | 成本失控 | 严格测试 + 监控告警 |
| 多语言翻译质量 | 用户体验差 | 找母语者校对 |

### 6.2 商业风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 用户获取成本高 | 亏损 | 内容营销 + SEO |
| 付费转化率低 | 收入不足 | 优化试用体验 |
| 竞争对手 | 市场份额 | 突出差异化功能 |

### 6.3 法律风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| GDPR 合规 | 罚款 | 隐私政策 + 数据加密 |
