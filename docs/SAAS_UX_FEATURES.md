# SaaS 开箱即用功能规划

面向**普通工作人员、学生、研究人员、开发者**的线上 SaaS，目标：**开箱即用、操作简单、设置极少**。

---

## 1. 与 OpenClaw / OpenCode 功能对比

基于 `docs/DEV_PLAN_OPENCLAW_OPENCODE.md`、`REFERENCE_OPENCLAW_OPENCODE_*.md` 的对比摘要：

| 维度 | OpenClaw | OpenCode | X-Computer 现状 |
|------|----------|----------|-----------------|
| **定位** | 本地「数字员工」：多平台消息 + 浏览器自动化 + 心跳式主动执行 | 开源 AI 编程 Agent：终端/VS Code/桌面，专注代码 | AI 驱动整机：Web 桌面 + 办公应用 + 四域任务 + X 主脑 |
| **形态** | 本地进程，可无头/服务端 | CLI + VS Code + Desktop | 仅 Web 单页 |
| **开箱即用** | 需配置 agents、channels、API Key | 需配置 LLM、工作区 | 有默认 .x-config，但用户仍需理解「大模型配置」 |
| **Skill 生态** | 100+ SkillHub Skills，按来源分组、启用/安装/API Key UI | SKILL.md 发现 + skill 工具加载 | 有 Skill 发现与自安装，无精选推荐 |
| **多平台消息** | 13+ 渠道（WhatsApp、Telegram、Slack 等） | 无 | 邮件 + WhatsApp + 应用内通知 |
| **浏览器自动化** | 独立 Chrome 实例，网页抓取、填表 | 无 | 仅有 URL 展示，无自动化 |
| **一键安装** | 本地安装 | curl/npm/Homebrew 一条命令 | 需 clone + npm run dev |
| **首次体验** | 无明确引导 | 无 | 登录后直接进桌面，无新手向导 |
| **快捷任务** | 无预设模板 | 无 | 无快捷模板，需用户描述 |

## 2. 新增功能清单（开箱即用、操作简单）

### 2.1 零配置启动（P0）

| 功能 | 说明 | 实现要点 |
|------|------|----------|
| **服务端默认 LLM** | 注册后无需填写 API Key 即可对话 | 已在 R030 支持 .x-config 默认；SaaS 部署时配置 `llm_config` + 服务端 API Key，用户首次进入即用 |
| **隐藏高级配置** | 普通用户看不到「大模型配置」「MCP」「Skills」等复杂 Tab | 设置页增加「简洁模式」：仅显示语言、主题、订阅；高级设置折叠或需「开发者模式」开启 |
| **默认自动模式** | 新用户任务默认「自动执行」，不弹审批 | 已有 executionMode；确保新用户默认 `automatic`，避免「需要审批」造成困惑 |

### 2.2 首次体验与引导（P1）

| 功能 | 说明 | 实现要点 |
|------|------|----------|
| **新手向导** | 首次登录后 3～5 步引导：打开 AI 助手 → 发一条消息 → 查看任务时间线 | 前端 `useFirstTimeUser()`，localStorage 记录 `onboarding_done`；引导完成后不再显示 |
| **示例对话** | AI 助手空状态时展示「试试这些」：写周报、总结文档、解释代码、搜索资料 | ChatApp 空状态显示 4～6 个示例 prompt，点击即填入输入框 |
| **桌面快捷入口** | 桌面显著位置放「AI 助手」「X 主脑」图标，减少「不知道点哪」 | 已有；可增加「开始使用」浮动提示指向 AI 助手 |

### 2.3 快捷任务模板（P1）

| 功能 | 说明 | 实现要点 |
|------|------|----------|
| **场景模板** | 按用户类型预设：学生（写作业、查文献）、办公（周报、会议纪要）、研究（文献总结、数据分析）、开发（代码解释、单元测试） | 新增「快捷任务」面板或 ChatApp 内模板选择；选择后自动填充 prompt 并创建任务 |
| **一键操作** | 「总结这份文档」「把这段代码加上注释」等，选中内容后右键或工具栏一键触发 | 需与文件管理器/编辑器联动；可先做「从剪贴板/上传文件总结」简化版 |
| **模板库** | 10～20 个精选模板，按场景分类，支持中英文 | 配置化：`templates.json` 或 API 返回；前端展示卡片，点击即用 |

### 2.4 预设能力开箱即用（P1）

| 功能 | 说明 | 实现要点 |
|------|------|----------|
| **服务端内置 MCP** | 搜索、网页抓取等由服务端预配置，用户无需填写 | 已在 SAAS_DEFAULT_CONFIG 说明；部署时配置 `X_COMPUTER_MCP_SERVERS` 或 `mcp-servers.json`，fetch 等无 Key 的 MCP 直接可用 |
| **精选 Skill 推荐** | 试用/个人版展示 5～10 个「推荐 Skill」，一键安装 | 新增 API `GET /api/skills/recommended`；前端「推荐」Tab，点击安装；与 SkillHub 或自建精选列表对接 |
| **默认模型即用** | 用户不配置时，使用服务端默认模型（如 gpt-4o-mini） | R030 已支持；确保 SaaS 部署时 `defaultByModality` 完整，且服务端 Key 有效 |

### 2.5 简化操作流程（P2）

| 功能 | 说明 | 实现要点 |
|------|------|----------|
| **对话即任务** | 用户发消息后，自动判断「需要多步执行」则创建任务，无需用户点「创建任务」 | 已有 intent 分类；可增强：复杂意图自动走任务流程，简单问答直接回复 |
| **任务进度可见** | 任务执行时，在聊天窗口内展示「正在执行：步骤 2/5」等，减少跳转 | 已有 tool_start/tool_complete 事件；可增强 ChatApp 内嵌任务进度条 |
| **结果一键复制/导出** | 任务完成后，提供「复制结果」「导出为文档」按钮 | 任务详情页增加操作按钮；导出可调用 office.create_docx |
| **错误友好提示** | 配额不足、模型不可用时，提示「升级套餐」或「联系管理员」，而非技术错误码 | 前端统一处理 429、402 等，展示友好文案 + 升级入口 |

### 2.6 按用户类型的预设（P2）

| 用户类型 | 预设能力 | 说明 |
|----------|----------|------|
| **学生** | 文献搜索、作业辅助、笔记总结 | 推荐 Skill：学习类；模板：总结 PDF、写读后感 |
| **研究人员** | 文献分析、数据整理、报告生成 | 推荐 Skill：学术类；模板：文献综述、数据可视化 |
| **办公人员** | 周报、会议纪要、Excel 处理 | 推荐 Skill：办公类；模板：周报、会议纪要、表格整理 |
| **开发者** | 代码解释、单元测试、文档生成 | 推荐 Skill：编程类；模板：代码审查、生成测试、API 文档 |

实现：注册或设置中可选「我主要用来」→ 学生/研究/办公/开发；根据选择预填推荐模板与 Skill 列表。也可不做选择，用通用默认。

### 2.7 不优先做的复杂功能（对齐 OpenClaw/OpenCode 但非开箱即用）

| 功能 | 原因 |
|------|------|
| 浏览器自动化（browser.*） | 需 Playwright/Puppeteer，配置复杂，非普通用户刚需 |
| 多平台消息（Slack/Telegram 等） | 已有邮件+WhatsApp；扩展需逐渠道配置，增加复杂度 |
| Heartbeat 条件监控 | 高级用户场景，不适合「开箱即用」 |
| VS Code 扩展 / CLI | 多端形态，与「Web 开箱即用」不同路线 |
| Skill 状态 UI（启用/依赖/安装） | OpenClaw 式，偏开发者；先做「推荐 + 一键安装」即可 |
| LSP 集成 | 深度开发场景，非普通用户 |

## 3. 实施优先级建议

| 优先级 | 功能 | 目标用户价值 |
|--------|------|--------------|
| **P0** | 零配置启动（服务端默认 LLM、隐藏高级设置） | 注册即用，无需理解「API Key」「MCP」 |
| **P1** | 新手向导 + 示例对话 | 降低「不知道能做什么」的困惑 |
| **P1** | 快捷任务模板（10+ 模板，按场景） | 一键完成常见任务，减少描述成本 |
| **P1** | 服务端内置 MCP + 精选 Skill 推荐 | 搜索、抓取等能力开箱即有 |
| **P2** | 对话即任务、任务进度可见 | 操作更顺畅，减少跳转 |
| **P2** | 按用户类型预设（学生/研究/办公/开发） | 更贴合角色，提升首次匹配度 |
| **P2** | 错误友好提示、结果一键导出 | 体验收尾，减少挫败感 |
